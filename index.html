Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The meta element defines metadata such as the character set, desrcription, keywords, author and viewport.
  • The charset element sets our character encoding.
  • Child elements that are nested inside other elements called parent elements.
  • Using semantic HTML elements makes your HTML easier to read by other developers and the browser.
  • An Element refers to an entire element from opening tag to closing tag.
  • A tag refers only to what appears inside the angle brackets.
  • ul element refers to unordered lists.
  • ol element refers to ordered lists.
  • li element refers to list items.
  • When using headings, it is important to use heading tags in order.
  • Alt text is important for screen readers and other assistive technologies.
  • The footer usually contains the author, copyright contact, sitemap, and navigation.

CSS

  • CSS defines the style.
  • Declarations contain the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Git commands always start with git and are used to save and track versions of your code.
  • Git is used for tracking changes locally on your own computer.
  • GitHub is used to collaborate with others on a cloud-based service.
  • git add -A adds all changes to the next commit
  • git commit -m "commit message" describes what the commit contains; captures a snapshot of the currently staged changes
  • git pull origin main makes sure you have the latest version of the project on your local computer.
